# Project Architecture - WAIBOT Pro

This repository follows a professional JavaScript/Node.js structure to ensure maintainability, security, and scalability.

## 📁 Directory Structure

- `src/`: Core application logic
    - `bot/`: WhatsApp client initialization (`client.js`) and event handlers.
    - `api/`: External service integrations (`ia.js` - DeepSeek/OpenAI).
    - `logic/`: Business logic handlers:
        - `interacao.js`: Main message processing flow.
        - `simulacao.js`: Consórcio simulation logic.
        - `followup.js`: Scheduled re-engagement timers.
        - `contorno_objecoes.js`: Objection handling logic.
        - `historias_sucesso.js`: Success story database.
    - `utils/`: Common helpers (`whatsapp.js` with `sendMessage` logic).
- `backend/`: Server-side API, dashboard routes, and database management.
- `frontend/`: React application for the administrative dashboard.
- `docs/`: Project documentation and guides.
- `tests/`: Automated tests (Unit/Integration).
- `scripts/`: Maintenance and helper scripts.

## 🛡️ Anti-Spam & Security Hardening

To protect your WhatsApp account from temporary or permanent bans ("detecção de spam"):

### 1. Whitelist Mode (Safe Mode)
- Defined in `index.js` via `MODO_SEGURO`.
- While enabled, the bot will **only respond to the `ADMIN_NUMBER`**.
- This prevents the bot from answering all current chats when you connect the account for testing.

### 2. Human Behavior Simulation
- Located in `src/utils/whatsapp.js`.
- **Random Delay**: Adds a mandatory 1-3 second pause before replying.
- **Typing Status**: Automatically sends the "Typing..." state to WhatsApp during the delay, making the bot appear human.

### 3. Filters
- Automatically ignores `status@broadcast` and empty messages.
- Prevents loops by ignoring messages from the bot itself (`fromMe`).

## ⚙️ Configuration

Ensure your `.env` is correctly set up from `.env.example`:
- `DEEPSEEK_API_KEY`: Required for AI responses.
- `JWT_SECRET`: Used for dashboard authentication.
- `ADMIN_NUMBER`: The number allowed in `MODO_SEGURO`.
